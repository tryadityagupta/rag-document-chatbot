# RAG Document Chatbot

A Retrieval-Augmented Generation chatbot that answers questions about a PDF, grounded only in retrieved passages. Built with a FastAPI backend and a lightweight web UI that shows the exact source chunks behind every answer.

## How it works

1. **Ingest** — the PDF is loaded, split into overlapping chunks, embedded with OpenAI `text-embedding-3-small`, and stored in a FAISS index.
2. **Retrieve** — each question is embedded and matched against the index to pull the top-k most relevant chunks.
3. **Answer** — the retrieved chunks are passed to `gpt-4o-mini` with a strict "answer only from context" prompt, so the model won't invent facts beyond the document.

The web UI surfaces the retrieved chunks under each answer, so you can verify exactly what the response was built from.

## Tech stack

- **Backend:** FastAPI, served with Uvicorn
- **Retrieval:** FAISS vector store
- **Models:** OpenAI `text-embedding-3-small` (embeddings), `gpt-4o-mini` (generation)
- **Orchestration:** LangChain
- **Frontend:** single static HTML page (vanilla JS)

## Project structure

```
rag-document-chatbot/
├── api.py            # FastAPI app: /chat endpoint + serves the UI
├── ingest.py         # builds the FAISS index from a PDF
├── static/
│   └── index.html    # chat interface
├── rag/
│   ├── loader.py     # PDF loading
│   ├── splitter.py   # chunking
│   ├── embeddings.py # OpenAI embeddings
│   ├── vectorstore.py# FAISS build/save/load
│   └── llm.py        # gpt-4o-mini
├── data/             # source PDF(s)
└── requirements.txt
```

## Setup

```bash
pip install -r requirements.txt
```

Create a `.env` file in the project root with your OpenAI key:

```
OPENAI_API_KEY=sk-your-key-here
```

## Usage

Build the index from your PDF (set the path in `ingest.py`):

```bash
python ingest.py
```

Start the server:

```bash
uvicorn api:app --reload
```

Then open **http://127.0.0.1:8000**.

## Notes

- The FAISS index in `vectorstore/` is generated by `ingest.py` and is git-ignored — rebuild it locally after cloning.
- Answers are constrained to the document. Out-of-context questions return "I don't know" by design.
